Criar projeto no Supabase
Ação manual — ~3 min
1
Acesse supabase.com e clique em Start your project. Faça login com sua conta GitHub ou Google (gratuito).
2
Clique em New project. Escolha um nome para o projeto (ex: yahoo-itom) e defina uma senha do banco (guarde ela).
3
Escolha a região South America (São Paulo) para menor latência. Clique em Create new project e aguarde ~2 minutos.
4
No painel do projeto, vá em Authentication → Settings e desative a opção "Enable email confirmations" para facilitar o cadastro de usuários.
💡 O plano gratuito do Supabase inclui 500 MB de banco, usuários ilimitados, 2 projetos e 50.000 requisições/mês — mais que suficiente para o sistema.
🛠️
Criar as tabelas do banco
SQL Editor — ~2 min
1
No painel do seu projeto Supabase, clique em SQL Editor no menu lateral.
2
Clique em New query, cole o código abaixo completo e clique em Run (▶).
📋 Copie e cole no SQL Editor do Supabase:
-- Tabela de perfis dos usuários create table public.profiles ( id uuid references auth.users primary key, email text, role text default 'user', created_at timestamptz default now() ); -- Planilha individual de cada usuário create table public.sheets ( user_id uuid references auth.users primary key, rows jsonb default '[]', updated_at timestamptz default now() ); -- Planilha Base (modelo do admin) create table public.template ( id int primary key default 1, rows jsonb default '[]', fixed jsonb default '[]', updated_at timestamptz default now() ); -- Habilitar acesso autenticado (RLS) alter table public.profiles enable row level security; alter table public.sheets enable row level security; alter table public.template enable row level security; create policy "allow_auth" on public.profiles for all using (auth.role() = 'authenticated'); create policy "allow_auth" on public.sheets for all using (auth.role() = 'authenticated'); create policy "allow_auth" on public.template for all using (auth.role() = 'authenticated');
✅ Se aparecer "Success. No rows returned" no SQL Editor, funcionou corretamente.
